# 常见模块

这一页整理训练营里常见的通用模块，方便后面综合项目引用。

## 1. 常见模块

| 模块 | 常见接口 | 入门实验 | 注意事项 |
| --- | --- | --- | --- |
| OLED 显示屏 | I2C | 显示计数和传感器数据 | 确认地址常见为 `0x3C` 或 `0x3D` |
| 继电器模块 | 数字输出 | 控制低压灯具 | 确认高低电平触发方式 |
| 蜂鸣器模块 | 数字输出或 PWM | 提示音和报警 | 区分有源、无源蜂鸣器 |
| 按键模块 | 数字输入 | 切换工作模式 | 使用上拉并处理抖动 |
| 电位器模块 | ADC | 调节亮度或阈值 | 输出电压不能超过 3.3V |
| 红外接收模块 | 数字输入 | 遥控器控制 | 记录遥控器按键编码 |
| 舵机模块 | 控制脉冲 | 调整角度 | 独立供电并与 ESP32 共地 |

## 2. 使用原则

1. 先确认电压范围。
2. 先确认引脚定义。
3. 先确认是否需要上拉或下拉。
4. 先确认驱动方式，再接到主程序里。

## 3. 单模块记录模板

每接入一个模块，至少记录下面这些信息，避免综合项目阶段重新查找：

```text
模块名称：
芯片或型号：
工作电压：
接口类型：
ESP32 引脚：
库名称和版本：
I2C 地址或通信参数：
最小示例是否通过：
异常现象和解决方法：
```

## 4. 接入主程序的步骤

1. 单独运行模块示例，确认硬件可用。
2. 把引脚、地址和阈值放入集中配置。
3. 用函数封装初始化、读取或控制操作。
4. 在主循环中使用 `millis()` 安排更新频率。
5. 打印关键状态，确认模块间没有引脚或资源冲突。
6. 断电重启后再次验证，不只测试热重启。

## 5. 常见冲突

- 两个 I2C 模块地址相同，需要改地址或使用多路复用器。
- SPI 设备可共用时钟和数据线，但每个设备需要独立 CS。
- PWM 通道、定时器或中断资源可能被多个库同时占用。
- 经典 ESP32 的部分 ADC 引脚在 Wi-Fi 工作时不可用。
- 启动相关引脚被外部模块拉到错误电平，可能导致无法开机或上传。
