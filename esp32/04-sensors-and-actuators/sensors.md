# 传感器

传感器是 ESP32 项目中最常见的数据来源。训练营前期最适合从几个典型类别开始学。

## 1. 常见传感器类型

### 1.1. 环境类

- DHT11
- DHT22
- BMP280
- BME280

### 1.2. 距离类

- HC-SR04
- VL53L0X

### 1.3. 姿态类

- MPU6050

### 1.4. 光电类

- 红外避障
- 光敏传感器

## 2. 先看接口再接线

| 接口 | 典型模块 | 接线重点 |
| --- | --- | --- |
| 数字输入 | 红外避障、人体感应 | 确认输出电平不超过 3.3V |
| 模拟输入 | 光敏、电位器、土壤湿度 | 输出必须在 ADC 允许范围内 |
| 单总线 | DHT11、DHT22 | 使用对应库，注意上拉和读取间隔 |
| I2C | BME280、MPU6050、VL53L0X | 确认地址、SDA、SCL 和上拉 |
| UART | GPS、部分空气质量模块 | 交叉连接 TX/RX，并共地 |

ESP32 的 GPIO 通常只接受 3.3V 电平。模块即使使用 5V 供电，也不代表它的信号输出能直接接入 ESP32。

特别注意：常见 HC-SR04 的 `Echo` 可能输出 5V，接入 ESP32 前应使用分压或电平转换。

## 3. 模拟光线读取示例

下面示例用经典 ESP32 的 `GPIO 34` 读取模拟量。其他型号的 ADC 引脚不同，应按开发板引脚图调整。

```cpp
const int sensorPin = 34;

void setup() {
  Serial.begin(115200);
  analogReadResolution(12);
}

void loop() {
  int rawValue = analogRead(sensorPin);

  Serial.print("raw=");
  Serial.println(rawValue);
  delay(200);
}
```

默认 12 位读数通常在 `0` 到 `4095` 之间。它只是原始值，不会自动变成亮度、距离或温度。真正的项目还要做标定、单位转换和异常值处理。

使用 Wi-Fi 时，经典 ESP32 应优先选择 ADC1 对应引脚，因为 ADC2 可能与 Wi-Fi 资源冲突。

## 4. 推荐实验顺序

1. 先读单个传感器。
2. 再看串口输出。
3. 记录静止、最小和最大三组数据。
4. 再把数值转成实际意义。
5. 设置阈值，并观察阈值附近是否抖动。
6. 最后接入执行器做联动。

## 5. 数据处理要点

1. 连续读取多次并取平均值，可以减小随机波动。
2. 开启和关闭使用不同阈值，可以避免执行器在临界值附近频繁切换。
3. 读取失败时应保留错误状态，不要把错误值当作真实数据。
4. 记录采样时间，网络上传时才能判断数据是否过期。

## 6. 排错清单

1. 核对模块供电电压和信号电平。
2. 确认 GND 已共地。
3. 用最小示例验证模块，不要一开始就接入完整项目。
4. I2C 模块先运行地址扫描程序。
5. 串口同时打印原始值和转换后的值。
