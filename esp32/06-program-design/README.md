# 程序设计

到了这一阶段，课程重点从“会接线、会控制”转向“会组织程序”。这一步很重要，因为项目一复杂，真正决定质量的通常不是单个函数，而是整体结构。

## 1. 本阶段内容

### 1.1. 学习顺序

1. [Millis 计时](./millis-timing.md)
2. [状态机](./state-machine.md)
3. [项目结构](./project-structure.md)
4. [调试方法](./debugging.md)

### 1.2. 学习目标

完成这一阶段后，学习者应该能够：

1. 用 `millis()` 写非阻塞逻辑
2. 用状态机管理复杂流程
3. 把一个项目拆成多个文件和模块
4. 建立清晰的调试流程
5. 为后面的综合项目做结构准备

## 2. 课程重点

### 2.1. 非阻塞代码

尽量减少 `delay()` 的使用。

### 2.2. 状态机

把复杂流程拆成多个可管理状态。

### 2.3. 模块化

让一个项目能继续扩展，而不是越写越乱。
