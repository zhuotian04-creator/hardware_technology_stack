# 项目检查清单

项目检查清单用于在训练营中快速确认一个作品是否已经具备可演示、可提交、可复现的基础条件。

## 1. 上电前

- [ ] 电源电压、极性和额定电流符合要求
- [ ] ESP32 与外部电源、模块已经共地
- [ ] 5V 信号没有直接接入 3.3V GPIO
- [ ] 电机、舵机、继电器等负载使用合适的驱动和电源
- [ ] 接线与方案表一致，没有松动或短路
- [ ] 启动相关引脚不会被外部模块拉到错误电平

## 2. 单模块

- [ ] 程序可以稳定编译和上传
- [ ] 串口波特率正确，启动日志清晰
- [ ] 每个传感器都通过独立读取测试
- [ ] 每个执行器都通过独立控制测试
- [ ] I2C 地址、GPIO 和通信参数已经记录
- [ ] 原始值、单位和有效范围符合预期

## 3. 系统联动

- [ ] 核心流程满足 [项目需求](./project-requirements.md) 中的验收条件
- [ ] 输入变化能触发正确输出
- [ ] 阈值附近不会频繁抖动
- [ ] 多个任务同时运行时没有明显卡顿
- [ ] Wi-Fi 或 MQTT 断开后，本地核心功能仍按设计工作
- [ ] 网络恢复后可以自动重连
- [ ] 传感器失败时不会误触发危险动作
- [ ] 执行器设置了最大运行时间或安全状态

## 4. 稳定性

- [ ] 完整断电再上电后能自动启动
- [ ] 连续运行时间达到项目要求
- [ ] 连续操作按钮、网页或命令不会卡死
- [ ] 电机或舵机动作时 ESP32 不会重启
- [ ] 无网络、错误输入和模块断开均已测试
- [ ] 串口没有持续刷新的异常或重复重连信息

## 5. 提交材料

- [ ] README 说明项目目标、功能和使用方法
- [ ] 提供清晰的接线图或引脚表
- [ ] 列出开发板、模块、库名称和版本
- [ ] 不包含真实 Wi-Fi 密码、令牌或私钥
- [ ] 新环境按文档能够重新编译和上传
- [ ] 保留关键串口日志或测试记录
- [ ] 准备一段稳定、可重复的演示流程

## 6. 最低通过条件

项目至少应满足：

1. 无明显供电和接线风险。
2. 核心功能可以从上电开始完整演示。
3. 一个模块失败时，设备不会进入危险状态。
4. 代码和接线说明能让另一位同学复现。
5. 仓库中没有敏感信息。

## 7. 使用方式

1. 开始前先列清单。
2. 调试时逐条打勾。
3. 交付前再完整复查一遍。
4. 未通过的项目要记录原因和下一步，不要只写“有问题”。
