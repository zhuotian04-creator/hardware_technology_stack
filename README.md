# Hardware Technology Stack

这是一个用于整理硬件技术栈的资料仓库，内容覆盖嵌入式开发、物联网硬件、开发工具链、PCB 设计、项目代码、调试记录和课程资料。

仓库的核心目标是把零散的硬件学习资料和项目经验整理成可检索、可复用、可持续维护的知识库，方便后续学习、教学、训练营组织和项目开发。

## 1. 仓库用途

本仓库主要用于存放：

- 嵌入式开发学习资料
- ESP32、STM32 等开发板相关文档
- 传感器、执行器和通信模块使用记录
- 物联网项目代码与实验记录
- PCB 设计资料与工程文件
- 开发环境、工具链和调试流程文档
- 硬件项目需求、检查清单和复盘资料

## 2. 目录说明

```text
hardware_technology_stack/
├── docs/      通用文档、补充资料和跨项目说明
├── esp32/     ESP32 学习资料、课程文档和项目示例
├── pcb/       PCB 设计资料、规范和工程文件
├── stm32/     STM32 学习资料、外设实验和项目代码
├── tools/     开发工具链安装、配置和使用笔记
└── README.md  仓库总说明
```

各目录应尽量保持内容独立。新增目录或重要文件时，建议同步更新对应目录的 `README.md`，并在本文件中补充入口说明。

## 4. 资料组织原则

1. 一个目录只负责一类主题，避免把课程、代码、工具说明和项目资料混在一起。
2. 重要目录应有独立的 `README.md`，说明该目录的用途、内容结构和阅读顺序。
3. 项目代码应包含硬件连接、依赖库、开发环境和运行方式说明。
4. 调试记录应尽量写清楚问题现象、排查过程、解决方式和结论。
5. 文档链接使用相对路径，保证在 GitHub 页面中可以直接跳转。

## 5. 命名规范

| 对象 | 推荐格式 | 示例 |
|---|---|---|
| 文件夹 | Kebab Case | `esp32-sensor-test` |
| 文本文档 | Kebab Case | `serial-debug-notes.md` |
| 媒体文件 | Kebab Case | `wiring-diagram.png` |
| 代码文件 | Snake Case | `sensor_reader.cpp` |
| 类名 | Pascal Case | `SensorReader` |
| 函数名 | Snake Case | `read_sensor_value()` |
| 常量 | Constant Case | `MAX_RETRY_COUNT` |

补充约定：

- 文档资料可以使用中文命名，但建议保持简短明确。
- 代码文件尽量使用英文命名，避免空格和特殊符号。
- 同一目录内的命名风格应保持一致。

## 6. 提交建议

提交信息建议简短、明确，能看出本次改动范围。

```text
docs: update repository structure
feat: add esp32 rgb demo
fix: correct platformio notes
chore: organize pcb files
```

提交前建议检查：

1. 文件是否放在合适目录。
2. 文档链接是否有效。
3. 示例代码是否能编译，或已注明验证状态。
4. 临时文件、编译缓存和本地配置是否没有误提交。
